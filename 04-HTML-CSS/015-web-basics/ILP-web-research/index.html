1. HTTP-Header in der Praxis

Bei der Analyse der Netzwerk-Anfragen wurde deutlich, dass Header essenzielle Metadaten für die Kommunikation liefern. Der User-Agent übermittelt Details zum Browser, während Accept-Language die bevorzugte Sprache des Nutzers klärt. Zusätzlich steuert Cache-Control die Effizienz durch Speicherregeln, und Content-Type definiert das Datenformat wie HTML oder JSON.

2. Der TLS-Handshake (HTTPS-Sicherheit)

Der TLS-Handshake sichert die Verbindung, indem Client und Server zunächst Verschlüsselungs-Standards im "Hello"-Austausch aushandeln. Anschließend authentifiziert sich der Server durch ein digitales Zertifikat, woraufhin beide Parteien einen kryptografischen Schlüssel für die Sitzung generieren. Erst nach diesem Prozess wird das erste verschlüsselte Datenpaket übertragen, was Integrität und Vertraulichkeit garantiert.

3. Die Evolution: Von HTTP/1.1 zu HTTP/2

HTTP/1.1 leidet unter dem "Head-of-Line Blocking", bei dem eine langsame Anfrage alle nachfolgenden Datenpakete in der Warteschlange blockiert. HTTP/2 löst dieses fundamentale Performance-Problem durch Multiplexing, wodurch mehrere Anfragen gleichzeitig über eine einzige TCP-Verbindung fließen können. Diese Technik reduziert Ladezeiten massiv, da der Browser nicht mehr auf die Fertigstellung einzelner Dateien warten muss.

Semantische Synthese

In diesem Dokument wurden gezielt <strong> für inhaltliche Relevanz und <em> für die sprachliche Betonung eingesetzt. Im Gegensatz zu rein visuellen Tags wie b oder i ermöglichen diese eine barrierefreie Interpretation durch Screenreader. Auf Layout-Hacks wie doppelte Zeilenumbrüche wurde verzichtet, da Abstände sauber über CSS-Eigenschaften definiert werden sollten.
